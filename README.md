# Crm_Bot

Discord-бот для учёта пропусков, отработок и управления отпусками удалённой команды.

Сотрудник пропустил работу — записал пропуск. Отработал позже — записал отработку.
Бот считает баланс: кто должен, у кого переработка. Всё по Московскому времени.

---

## Возможности

### Учёт рабочего времени
- **Пропуск** — запись пропущенных часов (не более 8 ч/день)
- **Отработка** — запись отработанных часов (не более 16 ч/день)
- **Статус** — полная сводка: все долги, отработки и итог с округлением до 0.01
- **Должники** — список тех, кто ещё не закрыл долги
- **История** — все записи сотрудника с фильтром по месяцам
- **Экспорт в CSV** — отчёт за любой период
- **Удаление** — своих записей (и чужих для админа)
- **Обнуление** — сброс переработок у всех с полным отчётом

### Управление отпусками
- **Заявка на отпуск** — пользователь отправляет, админ подтверждает через кнопки
- **Изменение/удаление отпуска** — заявка на перенос или отмену
- **График отпусков отдела** — `мои отпуска` показывает отпуска твоего отдела
- **Сводка по отделам** — для админа: `статус отпусков` группирует по подразделениям
- **Поздравления** — каждое утро в 9:30 МСК бот объявляет в общем канале, кто уходит в отпуск
- **Отделы** — определяются автоматически по тегу в нике: `[QA]`, `[DEV]`, `[PM]` и т.д.

### Общее
- **Админы** — назначение через секретный ключ или суперадмином
- **Поиск** — по @пингу, ID, имени пользователя или нику на сервере
- **Даты** — `ДД.ММ.ГГГГ`, `сегодня`, `вчера`, `завтра`
- **Время** — Московское (UTC+3)
- **Реакции** — ✅ успех, ❌ ошибка с пояснением
- **Актуальные имена** — всегда показывает текущий ник с сервера, а не тот, что был при создании записи

## Команды

### В общем канале (через @Crm_Bot)

| Команда | Описание |
|--------|----------|
| `@Crm_Bot пропуск 4` | Пропуск на сегодня, 4 часа |
| `@Crm_Bot пропуск 05.06.2026 2.5` | Пропуск на конкретную дату |
| `@Crm_Bot пропуск вчера 8` | Пропуск словами |
| `@Crm_Bot отработка 3` | Отработка на сегодня |
| `@Crm_Bot отработка завтра 6` | Отработка словами |
| `@Crm_Bot удалить пропуск 05.06.2026` | Удалить свою запись |
| `@Crm_Bot удалить отработка 05.06.2026 @User` | Удалить чужую запись *(админ)* |

### В личке (текстом)

| Команда | Описание |
|--------|----------|
| `статус` | Своя сводка |
| `статус @User` | Чужая сводка *(админ)* |
| `статус 123456789` | Поиск по ID *(админ)* |
| `статус Иван` | Поиск по имени или нику *(админ)* |
| `должники` | Список должников *(админ)* |
| `история @User` | Записи за 1 месяц *(админ)* |
| `история @User 3` | Записи за 3 месяца *(админ)* |
| `уволить @User` | Удалить все записи *(админ)* |
| `снять @User` | Снять админа *(админ)* |
| `отчёт @User` | CSV за 1 месяц *(админ)* |
| `отчёт @User 6` | CSV за 6 месяцев *(админ)* |
| `new admin @User` | Назначить админа *(админ)* |
| `обнулить` | Сбросить переработки у всех *(админ)* |
| `стать админом ключ` | Получить права админа |
| `отпуск ДД.ММ.ГГ-ДД.ММ.ГГ` | Отправить заявку на отпуск |
| `отпуск ДД.ММ.ГГ-ДД.ММ.ГГ @User` | Назначить отпуск *(админ)* |
| `изменить отпуск ДД.ММ.ГГ-ДД.ММ.ГГ на ДД.ММ.ГГ-ДД.ММ.ГГ` | Перенос отпуска |
| `изменить отпуск ДД.ММ.ГГ-ДД.ММ.ГГ на 0` | Удалить отпуск |
| `мои отпуска` | График отпусков своего отдела |
| `статус отпусков` | Сводка по всем отделам *(админ)* |

### Слеш-команды

| Команда | Где | Описание |
|--------|-----|----------|
| `/help` | Везде | Правила работы. В общем канале — базовые, в личке админ видит всё |
| `/отпуск` | Везде | Справка по отпускам (в общем канале — инструкция в личку) |

## Лимиты

| Тип | Максимум в день |
|-----|-----------------|
| Пропуск | 8 часов |
| Отработка | 16 часов |

Часы — дробные, через точку: `2.5`, `1.25`.

## Роли

| Роль | Как получить | Права |
|------|-------------|-------|
| Суперадмин | Прописан в коде (`config.py`) | Всё, включая `new admin` и `снять` |
| Админ | `стать админом ключ` или `new admin @User` | Все команды |
| Пользователь | По умолчанию | Пропуск, отработка, удаление своих, статус, отпуска |

## База данных

SQLite, 6 таблиц:

| Таблица | Назначение |
|---------|------------|
| `absences` | Пропуски |
| `overtimes` | Отработки |
| `log` | Логи действий |
| `admins` | Админы (кроме суперадминов) |
| `history_log` | История удалений и обнулений |
| `vacations` | Отпуска (только user_id, даты и статус) |

Имена пользователей **не хранятся** в базе — всегда подтягиваются с сервера по `user_id`. Отдел определяется по тегу `[TAG]` в нике.
